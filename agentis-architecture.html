Core concept

What Agentis does

Agentis is a skill-orchestration layer on top of the Claude API. Instead of sending one monolithic prompt, it chains multiple specialised "skill" sub-prompts in sequence (or as a DAG), so the model behaves like a team of experts rather than a generic chatbot. The core idea comes from obra/superpowers: small, composable, opinionated skills stacked together produce dramatically better output.

User picks persona → Mode chosen → Skill graph assembled → Claude API called per skill → Output rendered → Export

Execution engines (how skills are called)
FREEFORM · src/lib/claude.ts

Serial Pipeline

buildPipeline() selects steps by personaId. runAgentStreaming() loops through steps in order, accumulating previous outputs as context for each next call. Each step is one Claude API call with a skill system prompt.

TEMPLATE · src/lib/engine.ts

DAG Workflow Engine

runWorkflow() resolves a dependency graph. Nodes whose dependsOn[] are all satisfied run concurrently via Promise.all(). Critique nodes get the output of their target via critiques[].

OPENFANG · src/lib/openfang-runner.ts

External Agent Runtime

buildOpenFangPipeline() creates a stub pipeline displayed in the UI. runOnOpenFangHand() POSTs the task + personaId to the OpenFang daemon at localhost:4200. Streams SSE events back and maps them to the same step_start / step_stream / step_done / all_done chunk interface.

Library layer (src/lib/)

claude.ts

buildPipeline() — creates serial skill steps per persona.
runAgentStreaming() — calls Claude API once per skill, passes previous outputs as context, emits SSE-style chunks to UI.

engine.ts

runWorkflow() — resolves DAG, runs ready nodes concurrently, collects outputs, extracts artifacts, emits workflow events. Direct Claude API calls with streaming.

prompts.ts

SKILL_PROMPTS map — 18 skill prompt functions. Each takes (task, depsContext, critiqueContext) and returns {system, user}. buildNodePrompt() dispatches to these.

artifacts.ts

extractArtifacts() — parses markdown output for fenced code blocks and tags them as artifacts (code, yaml, sql, markdown). Used by OutputScreen / ArtifactViewer.

exportJob.ts

downloadJobZip() — packages skill outputs + execute.sh into a JSZip archive. User downloads it and runs it with Claude Code.

exportOpenFang.ts

exportFreeformToOpenFang() — POSTs freeform job to OpenFang daemon. Used in LegacyOutputScreen "Send to OpenFang" button.

openfang-runner.ts

buildOpenFangPipeline() + runOnOpenFangHand() — live execution path. Streams from OpenFang daemon and maps events to the standard chunk interface.

colors.ts

SKILL_COLORS map + DEFAULT_SKILL_COLOR. Maps skill name strings to {bg, border, text} CSS colour tokens. Used to colour nodes and pills consistently.

External integrations & deployment

Anthropic Claude API

Direct browser access via anthropic-dangerous-direct-browser-access: true header. Model: claude-sonnet-4-20250514. Streaming via SSE (stream: true). API key stored in React state only — never persisted.

OpenFang

Local agent runtime at localhost:4200. Installed via bin/openfang + install.sh. Dev script (npm run dev) starts both Vite and the OpenFang daemon concurrently via scripts/openfang-daemon.mjs.

Vercel Deployment

vercel.json config. VITE_ANTHROPIC_API_KEY env var. If not set, app shows the key-entry screen at runtime — safe for public deployment. dist/ folder contains built output including landing page.

How to extend

Add a new Skill

1. Add entry to SKILLS in src/data/skills.ts with colour tokens.
2. Add a prompt function to SKILL_PROMPTS in src/lib/prompts.ts.
3. Reference the skill ID in a persona's skills[] array.

Add a new Persona

Add an object to PERSONAS in src/data/personas.ts with an id, label, icon, description, skills[] array of skill IDs, and suggestions[] for task prompts.

Add a new Template

Add a WorkflowTemplate to WORKFLOW_TEMPLATES in src/data/templates.ts. Define nodeDefinitions[] with dependsOn[] for DAG edges, critiques[] for adversarial review nodes, and parallelGroup for visual grouping.
