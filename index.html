Features

Built for real work,
not demos

Every feature is designed around one principle: make the agents' work visible and trustworthy.

⬡
Agent Universe Canvas
Interactive hexagonal canvas showing each agent's role, provider, live output, token count, and dependencies in real time. Watch the orchestration graph build itself.
⚡
Smart Failover
If a provider fails mid-stream, agents automatically switch to the next available one. Zero data loss — output accumulates seamlessly across provider switches.
🔍
Live Web Search
Researcher and analyst agents auto-inject Tavily web search results into their context before generating output. Real-time knowledge, not just training data.
🧠
Persistent Universe
Ask follow-ups and the relevant agents are recalled with their full context. New agents spawn for what's new. The knowledge graph compounds across every turn.
📦
skills.sh Integration
Search 90,000+ skills from the skills.sh registry. Install and assign them to specific agent roles — specialized procedural knowledge injected at runtime.
🌐
Browser Agent
A dedicated agent autonomously navigates live websites using PinchTab — navigates, reads, clicks, and fills forms to gather real-time data from the web.

How it works

From one sentence
to a full agent team

Three phases. Fully automatic. Completely visible.

01
Plan
The orchestrator analyzes your task, breaks it into 2–12 sub-tasks, assigns roles and providers, and maps out which agents can run in parallel vs. sequentially.
02
Execute
Parallel agents fire simultaneously. Dependent agents receive upstream context and execute in order. Every output streams live. Failover triggers automatically.
03
Synthesize
The orchestrator merges all outputs into one cohesive answer. Ask a follow-up and the universe extends — existing agents recalled, new ones spawned as needed.
